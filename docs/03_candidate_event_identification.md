# 阶段三：候选事件识别（验收版）

## 1. 阶段目标

本阶段仅识别可能需要在后续阶段进一步理解和标注的资本事件，不输出最终 PE/VC 结论，不计算投资金额、每股价格或持股比例，也不重建完整股权结构。

## 2. 输入

唯一主要事实来源：

```text
301563_云汉芯城_IPO招股说明书.pdf
```

沿用阶段一、阶段二已验收的基础：

- PDF 阅读器页码 = 正文印刷页码 + 1；
- C2、C3 为核心扫描范围；
- C7、C8 为必要辅助核对范围；
- C2-3 为 PDF 第 56—58 页 / 正文第 55—57 页的连续跨页流程图；
- 流程图必须采用连续页面视觉核对，不能只依赖线性文本。

## 3. 识别规则

出现下列信号时建立候选事件：公司设立、整体变更、增资、股权或股份转让、股东进入或退出、对赌及其他特殊权利的签署、变更或解除。

同一时间段同时包含增资和股权转让的，标记为 `compound_event`，阶段三不拆分为逐笔交易。同一事件在流程图、正文标题或辅助章节中重复披露的，只保留一个稳定事件编号，并记录重复证据位置。

股东名单、发行前股本表和股东穿透表属于静态信息，不因出现某一股东名称而自动建立交易事件。否定性披露单独记录在 `auxiliary_findings.csv`。

## 4. 操作过程

1. 连续扫描 C2、C3 及 C7、C8 的必要辅助章节。
2. 对 PDF 第 56—58 页跨页流程图进行三页连续视觉识别。
3. 按纵向箭头顺序还原 13 个流程图节点。
4. 将流程图节点与 C3-1 至 C3-5 正文事件标题交叉比对并去重。
5. 将 C3-6 建立为对赌协议解除父级事件组，并按人工视觉确认建立 A、B、C、D、D+ 五个轮次级候选项。
6. 将 C3-7、C3-8、C3-9 和 C8-5 分别作为辅助披露、重复披露或否定性披露处理。
7. 固定正式候选事件编号，生成机器可读文件。

## 7. 辅助披露处理

- C3-7“股权代持”：否定性辅助披露，不建立资本事件。
- C3-8“交易价格无明显异常”：交易合理性辅助披露，不建立资本事件。
- C3-9“整体变更基准日未分配利润为负”：作为 CE-008 的补充证据，不建立第二个整体变更事件。
- C8-5明确披露“发行人不存在最近一年新增股东的情况”：不建立最近一年新增股东候选事件。该结论仅适用于招股书所称最近一年，不否定更早期间曾有股东进入。
- C7及C8其他静态股东信息：仅供后续主体身份和股权状态核对。

## 8. 阶段边界

本验收版没有执行以下工作：

- 未计算或校验增资金额、注册资本、股份数量及持股比例；
- 未计算每股价格或公司估值；
- 未重建任一时点的完整股权结构；
- 未判断任何主体是否属于 PE、VC、产业资本或其他投资者；
- 未判断对赌条款是否彻底解除或是否可能恢复；
- 未形成最终投资路径。

流程图和正文中的数值均属于原文披露值。可能存在的数值勾稽问题统一留到阶段七。
